# Getting Started

## Install

Install the two rxfy packages:

```bash
npm install rxfy rxfy-react
```

rxfy declares `rxjs`, `zod`, and `lodash` as **peer dependencies**: it expects your app to
provide them rather than bundling its own copies, so the whole app shares one instance of
each (one RxJS, one Zod). npm does not install peer dependencies automatically, so add them
yourself:

```bash
npm install rxjs zod lodash
```

Version ranges: `rxjs ^7`, `zod ^3`, `lodash ^4`. `rxfy-react` additionally needs
`react`/`react-dom` 18+ (which your React app already has), plus `next >=14` if you use the
`rxfy-react/next` subpath. In a TypeScript project, also add `@types/react` as a dev dependency:

```bash
npm install --save-dev @types/react
```

## Wrap your app

`StoreProvider` creates the model registry that `useStateData` and `useModelStore`
write to and read from.

```tsx
import { StoreProvider } from "rxfy-react";
import { createRoot } from "react-dom/client";

createRoot(document.getElementById("root")!).render(
  <StoreProvider>
    <App />
  </StoreProvider>,
);
```

A plain client-only app passes no props to `StoreProvider`. The SSR props
(`ssr`, `registry`, `dehydratedState`) are covered in
[Server-Side Rendering](/ssr).
