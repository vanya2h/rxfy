# React Bindings [Hooks and helpers for using rxfy in React]

`rxfy-react` is the official React binding for rxfy. Wrap your tree in `StoreProvider`
(see [Getting Started](/getting-started)), then reach for the hook you need:

- [`useStateData`](/react/use-state-data) — fetch, normalize, and subscribe to a query;
  returns a `StateHandle` (`data$`, `set`, `setRaw`, `reload`, `mutations`). Covers
  `defaultData` seeding and `set` vs `setRaw`.
- [`useStatePagedData`](/react/use-state-paged-data) — paginated / infinite-scroll lists of
  a single entity type, built on `useStateData`.
- [`useModelStore`](/react/use-model-store) — subscribe to one normalized entity by id.
- [`useAtom`](/react/use-atom) — two-way binding for any `IAtom` (a plain `Atom`, a `Lens`,
  or a `ModelStore.get(id)` cell).
- [`Pending`](/react/pending) — render pending / rejected / fulfilled UI for any observable.
- [`usePending`](/react/use-pending) — the status value behind `Pending`, when you need the
  `IWrapped<T>` rather than rendered UI.
- [`useObservable`](/react/use-observable) — bind a raw `Observable<T>` to React.
- [`createLiveClient`](/framework/client) — connect a WebSocket transport so server pushes land
  in the store. It lives in the framework-agnostic [`rxfy-client`](/framework/client) runtime and
  is re-exported here (alongside `readSsrGrants`); see
  [Sync Client in React](/react/live-client) for the `StoreProvider` wiring and the
  `updatesAvailable$` / `applyUpdates` handles.

`data$` from `useStateData` (and `useStatePagedData`) emits the **normalized query shape:
entity ids, not entities**. Render lists by id and read entity data through
[`useModelStore`](/react/use-model-store).
