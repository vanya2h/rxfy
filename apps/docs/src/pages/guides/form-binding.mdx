# Two-way form binding

Bind a form input to one field of a stored entity, so edits flow back into the model store
and reach every component showing that entity. This guide covers single fields, nested
fields, and a multi-field form.

The pieces: [`store.entity(id)`](/models-state#modelstore-entity) is a writable handle on
one entity, [`createLens`](/core-concepts/lens) focuses a field of it, and
[`useAtom`](/react#useatom-and-two-way-binding) binds that field to React as
`[value, setValue]`.

> The entity must already be in the store — `store.entity(id)` reads `undefined` for a key
> that was never `set`. Load it through `useStateData` (or a direct `store.set`) first.

## Bind one field

```tsx
import { useMemo } from "react";
import { createLens, keyLens } from "rxfy";
import { useAtom, useModelStore } from "rxfy-react";
import { TodoModel } from "./models";

function EditTitle({ id }: { id: string }) {
  const store = useModelStore(TodoModel);
  const todo$ = useMemo(() => store.entity(id), [store, id]); // IAtom<Todo>
  const title$ = useMemo(() => createLens(todo$, keyLens("title")), [todo$]);
  const [title, setTitle] = useAtom(title$);

  return <input value={title} onChange={(e) => setTitle(e.target.value)} />;
}
```

Each keystroke writes through the lens into the todo's cell. A list row, a detail header,
any subscriber of this todo updates in the same render — no re-fetch.

Wrap `store.entity(id)` and every `createLens` in `useMemo`. `useAtom` needs a stable
`IAtom` identity across renders; a handle rebuilt each render resets the binding.

## Bind a nested field

Lenses compose. To reach `address.city` on a user, stack a `keyLens` per level:

```tsx
const user$ = useMemo(() => store.entity(id), [store, id]);
const address$ = useMemo(() => createLens(user$, keyLens("address")), [user$]);
const city$ = useMemo(() => createLens(address$, keyLens("city")), [address$]);
const [city, setCity] = useAtom(city$);
```

Each `set` returns a new object up the chain — `setCity` produces a new `address`, then a
new `user`, written back to the store immutably. Nothing is mutated in place.

## A multi-field form

Give each input its own lens over the same entity handle. They share one source, so all of
them stay consistent:

```tsx
function EditTodo({ id }: { id: string }) {
  const store = useModelStore(TodoModel);
  const todo$ = useMemo(() => store.entity(id), [store, id]);

  const title$ = useMemo(() => createLens(todo$, keyLens("title")), [todo$]);
  const done$ = useMemo(() => createLens(todo$, keyLens("done")), [todo$]);

  const [title, setTitle] = useAtom(title$);
  const [done, setDone] = useAtom(done$);

  return (
    <form>
      <input value={title} onChange={(e) => setTitle(e.target.value)} />
      <input type="checkbox" checked={done} onChange={(e) => setDone(e.target.checked)} />
    </form>
  );
}
```

## Notes

- **Reads are live.** If the entity changes elsewhere — a mutation, a websocket push — the
  input re-derives from the new value. The lens is a window on the store, not a snapshot.
- **No-op writes are dropped.** Lens change detection uses `lodash.isEqual`, so setting a
  field to its current value won't echo back through subscribers.
- **Local draft state.** When you need an edit to apply only on submit (not on every
  keystroke), keep a local `useState` draft and write the entity once on submit with
  `store.set(id, draft)` — skip the lens binding for that field.
