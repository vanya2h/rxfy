# Normalization

Normalization is the idea rxfy is built around: store each entity **once**, keyed by its id,
and reference it by id everywhere else. This page explains what that means and why it matters;
[Models & State](/models-state) covers the API that does it.

## The problem: duplicated entities

A typical API response nests and **duplicates** entities. Here a post's `author` also left a
comment, so the same user is embedded twice:

```json
// denormalized â€” user "u1" (Alice) appears in two places
{
  "post": { "id": "p1", "title": "Hello" },
  "author": { "id": "u1", "name": "Alice" },
  "commenters": [
    { "id": "u1", "name": "Alice" },
    { "id": "u2", "name": "Bob" }
  ]
}
```

Duplicated copies drift apart: rename Alice and you have to find and patch every copy, and any
view still holding an old one is now wrong.

## Storing each entity once

**Normalizing** means storing each entity once, keyed by its id, in a per-model lookup table â€”
then referencing it by id everywhere else. rxfy does this on every fetch: `createModel` +
`getKey` define the id, `defineState` says which fields hold which model, and the fetch payload
is split into model stores (entities, by id) plus a **query shape** of ids:

```ts
// UserModel store â€” Alice stored once, shared by author + commenters
{ u1: { id: "u1", name: "Alice" }, u2: { id: "u2", name: "Bob" } }
// PostModel store
{ p1: { id: "p1", title: "Hello" } }

// what data$ emits â€” ids only
{ post: "p1", author: "u1", commenters: ["u1", "u2"] }
```

## Why it matters

Now Alice lives in exactly one place. Components render by id and subscribe to that single
entity, so one `userStore.set("u1", { ...alice, name: "Alicia" })` â€” from a refetch, a
mutation, or a websocket â€” updates the author byline **and** the comment in the same render,
with no re-fetch and no stale copies. That is why `data$` emits ids rather than entities.

Two entities of the same model with the same id are the same cell, even across different
queries: a user returned by a "current user" fetch and by a "post authors" list is stored
once, so both views stay in lockstep.

## In rxfy

- [`createModel`](/models-state#createmodel) defines an entity type and its id (`getKey`).
- [`defineState`](/models-state#definestate) declares which fields hold which model; the fetch
  result is normalized into stores + an id-only query shape automatically.
- [`data$`](/react#usestatedata) emits that id shape; [`useModelStore`](/react#usemodelstore)
  reads an entity by id and re-renders only when that entity changes.
