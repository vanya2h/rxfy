# rxfy

**rxfy** (/ɑɹ ɪks faɪ/) is a small library that lets you declare typed models and the states that query them, then access
their data as reactive observables. [Normalization](/core-concepts/normalization) keeps your app consistent
and reactive at no extra cost.

rxfy is built on three principles. Every entity lives in one slot, keyed by its id; a write
reaches every subscriber with no list re-fetch. Each slot is an RxJS Observable that
components subscribe to directly. You declare the fetch shape, the model, and the mutations;
rxfy normalizes the result, handles SSR, and rehydrates the client without a second fetch.
[Why rxfy?](/why) explains the thinking behind this design.

## Quick taste

**Step 1 — Define the model.**

A model binds a Zod schema to an id extractor. Every entity of this type lives in exactly one store slot, keyed by `getKey`.

```tsx
import { z } from "zod";
import { createModel } from "rxfy";

const TodoModel = createModel(z.object({ id: z.string(), title: z.string(), done: z.boolean() }), {
  getKey: (t) => t.id,
  name: "todo",
});
```

**Step 2 — Define state and mutations.**

`defineState` declares the fetch params, which models the response contains, and how mutations transform it. rxfy normalizes the result into model stores automatically.

```tsx
import { defineState, array } from "rxfy";

const todosState = defineState({
  key: "todos",
  params: z.object({ filter: z.enum(["all", "active", "done"]) }),
  model: { todos: array(TodoModel) },
  mutations: {
    addTodo: (prev, todo: { id: string; title: string; done: boolean }) => ({
      ...prev,
      todos: [...prev.todos, todo],
    }),
  },
});
```

**Step 3 — Fetch and render the list.**

`useStateData` wires the state to your fetch function and returns a reactive `data$` stream. `Pending` handles the loading and error states declaratively. The query shape carries only ids — not full entities.

```tsx
import { Pending, useStateData } from "rxfy-react";

function TodoApp() {
  const { data$, mutations } = useStateData({ state: todosState, fetchFn: fetchTodos, params: { filter: "all" } });
  return (
    <Pending
      value$={data$}
      pending={<p>Loading...</p>}
      rejected={({ error }) => <p>Something went wrong: {String(error)}</p>}
    >
      {({ todos }) => (
        <ul>
          {todos.map((id) => (
            <TodoItem key={id} id={id} />
          ))}
        </ul>
      )}
    </Pending>
  );
}
```

**Step 4 — Subscribe to individual entities.**

Each item subscribes to its own store slot. A single `store.set` — from a refetch, mutation, or websocket push — updates every component showing that entity, with no list re-fetch.

```tsx
import { useMemo } from "react";
import { useModelStore } from "rxfy-react";

function TodoItem({ id }: { id: string }) {
  const store = useModelStore(TodoModel);
  const todo$ = useMemo(() => store.get(id), [store, id]);
  return <Pending value$={todo$}>{(todo) => <li>{todo.title}</li>}</Pending>;
}
```
