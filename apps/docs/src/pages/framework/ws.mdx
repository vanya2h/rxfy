# rxfy-ws [The default WebSocket transport]

`rxfy-ws` ships two entry points: `rxfy-ws` for the server-side adapter and
`rxfy-ws/client` for the browser. Both speak [rxfy-protocol](/framework/protocol)
and wire into `rxfy-server`'s `Hub`.

## Server

`createWsServer(hub)` attaches a `Hub` to WebSocket connections. Call it once at
startup; call `handleConnection` for every new socket.

```ts
import { createWsServer } from "rxfy-ws";

type ServerSocket = {
  send: (data: string) => void;
  on: (event: string, listener: (...args: unknown[]) => void) => void;
};

function createWsServer(hub: Hub): { handleConnection: (socket: ServerSocket) => void };
```

`ServerSocket` is a structural type — the `ws` package's `WebSocket` satisfies it
directly. Frameworks that wrap the underlying socket (Hono, Bun, etc.) need a thin
adapter.

The Hono bridge from `examples/vite-blog-framework` shows the pattern: create an
`EventEmitter` to act as the event bus, pass a `ServerSocket` literal to
`handleConnection` on open, and relay `onMessage` / `onClose` events through the emitter.

```ts [server/ws.ts]
import { EventEmitter } from "node:events";
import type { UpgradeWebSocket } from "hono/ws";
import { createWsServer } from "rxfy-ws";
import { hub } from "./live.js";

const wsServer = createWsServer(hub);

export function liveRoute(upgradeWebSocket: UpgradeWebSocket) {
  return upgradeWebSocket(() => {
    const emitter = new EventEmitter();
    return {
      onOpen(_evt: Event, ws: { send: (data: string) => void }) {
        wsServer.handleConnection({
          send: (data: string) => ws.send(data),
          on: (event, cb) => emitter.on(event, cb),
        });
      },
      onMessage(evt: MessageEvent) {
        emitter.emit("message", evt.data);
      },
      onClose() {
        emitter.emit("close");
      },
    };
  });
}
```

## Client

`createWsClient` opens a WebSocket to `url` and returns a `ClientTransport` that
`createLiveClient` (from `rxfy-react`) consumes.

```ts
import { createWsClient } from "rxfy-ws/client";

type WsClientOptions = {
  url: string;
  /** Defaults to the global `WebSocket`. Inject for Node or tests. */
  WebSocketImpl?: (url: string) => WebSocketLike;
  /** Delay before each reconnect attempt. Default: 1000 ms. */
  reconnectDelayMs?: number;
};

type ClientTransport = {
  subscribe: (ids: string[]) => void;
  unsubscribe: (ids: string[]) => void;
  /** Single-slot inbound handler — a later call replaces the previous one. */
  onMessage: (handler: (message: ServerMessage) => void) => void;
  close: () => void;
};

function createWsClient(options: WsClientOptions): ClientTransport;
```

**Auto-reconnect and subscription replay.** The client keeps an internal set of
active subscription ids. On `"close"` it schedules a reconnect after
`reconnectDelayMs`; on `"open"` it immediately re-sends a subscribe frame for every
id in that set. Subscriptions made before the socket is `OPEN` are buffered and sent
once the connection is ready — no caller-side queuing needed.

```tsx [src/entry-client.tsx]
import { createModelRegistry } from "rxfy";
import { createLiveClient, readSsrGrants, StoreProvider } from "rxfy-react";
import { createWsClient } from "rxfy-ws/client";

const registry = createModelRegistry();
const liveClient = createLiveClient({
  registry,
  transport: createWsClient({
    url: `${location.protocol === "https:" ? "wss" : "ws"}://${location.host}/live`,
  }),
  grants: readSsrGrants(),
});

hydrateRoot(
  document.getElementById("root") as HTMLElement,
  <StrictMode>
    <StoreProvider registry={registry} ssr liveClient={liveClient}>
      <App url={location.pathname} />
    </StoreProvider>
  </StrictMode>,
);
```

## Bring your own transport

`ClientTransport` is a plain interface. `createWsClient` is the default
implementation, but anything that satisfies the four-method shape works — a
mock for tests, a custom reconnection policy, or a non-WebSocket channel.

```ts
type WebSocketLike = {
  readyState: number;
  send: (data: string) => void;
  close: () => void;
  addEventListener: (type: string, listener: (event: unknown) => void) => void;
  removeEventListener?: (type: string, listener: (event: unknown) => void) => void;
};
```

The wire format a custom transport must produce and consume is defined by
[rxfy-protocol](/framework/protocol).
