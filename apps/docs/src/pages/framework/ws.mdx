# rxfy-ws [The default WebSocket transport]

`rxfy-ws` ships two entry points: `rxfy-ws` for the server-side adapter and
`rxfy-ws/client` for the browser. Both speak the
[sync message protocol](/framework/server/messages) and wire into `rxfy-server`'s `Hub`.

- [`createWsServer`](/framework/ws/server) — attach a `Hub` to WebSocket connections,
  including the adapter pattern for frameworks that wrap the socket (Hono, Bun).
- [`createWsClient`](/framework/ws/client) — the browser transport with auto-reconnect
  and subscription replay.
- [Custom transports](/framework/ws/custom-transport) — the `ClientTransport` interface
  for test mocks, custom reconnection policies, or non-WebSocket channels.
