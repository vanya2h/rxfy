# live.grant [Mint the subscription allow-list]

`live.grant(registry, spec)` issues a `{ entities, channels }` token the client uses to subscribe
to live updates. Pass it back in the API response alongside the initial data.

```ts [server/api.ts]
const registry = createModelRegistry();
normalizeResult(registry, postsState.fields, data);

const grants = live.grant(registry, {
  entities: [postResource, userResource],
  states: [{ state: postsChannel, params: {} }],
});

return c.json({ data, grants });
```

- `entities` — the resources whose loaded rows should be grantable. Only ids actually present in
  the registry are included, keyed through
  [`createTopicKeyer`](/framework/server/create-topic-keyer).
- `states` — concrete state instances (`{ state: StateChannelDescriptor; params }`) whose
  invalidation channels the client may watch.

The returned `grants` object has shape
`{ entities: Record<string, string>; channels: Record<string, string> }` — a
topic-or-channel → hashed-key lookup the client uses when opening subscriptions.

For wiring grants into the client and hydrating a live session, see
[Grants & live hydration](/framework/server/grants).
