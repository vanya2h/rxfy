# rxfy-server [Bind Drizzle tables, write, and publish live updates]

`rxfy-server` bridges Drizzle ORM tables with rxfy models, handles database writes, and
publishes live update messages through a hub. Install it server-side only; `rxfy-server/browser`
re-exports the subset safe to import in shared (client+server) code — `defineResource`,
`createResourceRegistry`, `invalidationChannel`, and their types.

- [`defineResource`](/framework/server/define-resource) — tie a Drizzle `PgTable` to an rxfy
  model; `createResourceRegistry` indexes resources for server-side wiring.
- [`createServer`](/framework/server/create-server) — wire the database, resources, hub, and
  keyer into a `Live` object with typed write methods.
- [Writes](/framework/server/writes) — `live.create` / `live.update` / `live.delete`, plus
  `touch()` for state-channel invalidation.
- [`createTopicKeyer`](/framework/server/create-topic-keyer) — time-windowed HMAC ids that keep
  raw topic strings off the wire.
- [Grants & live hydration](/framework/server/grants) — mint the subscription allow-list with
  `live.grant` and flow it from an SSR render into a connecting live client, end to end.
- [`createInMemoryHub`](/framework/server/hub) — the single-process pub/sub backbone between
  writes and transports.

Start with [`defineResource`](/framework/server/define-resource) and
[`createServer`](/framework/server/create-server); for an end-to-end walkthrough see the
[Live blog guide](/guides/live-blog).
