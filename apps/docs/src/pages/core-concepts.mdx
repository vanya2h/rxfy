# Core Concepts [The ideas rxfy is built on]

Understanding-oriented pages: the principles behind rxfy and how the pieces fit together.
For the API surface of each primitive, see the [rxfy](/rxfy) package section.

- [Observables](/core-concepts/observables) — the pattern the rest of rxfy assumes: a value that
  changes over time and that you subscribe to. Read this first if RxJS is new to you.
- [Normalization](/core-concepts/normalization) — the principle behind rxfy: one keyed cell per
  entity, referenced by id everywhere else.
- [Late Unwrapping](/core-concepts/late-unwrapping) — why async state travels wrapped and unwraps
  only at the leaf that renders it, so re-renders stay narrow and writes never wait on reads.
- [Server-Side Rendering](/core-concepts/ssr) — how a per-request registry is dehydrated into
  the HTML and rehydrated on the client, in buffered, streaming, and two-pass modes.

To consume the primitives from React, see [React Bindings](/react).
