# Playwright Sync e2e Suite — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A private `examples/e2e` (`rxfy-e2e`) Playwright package that drives all six sync apps in a real browser and asserts live cross-tab sync, wired as a `turbo run e2e` gate in CI — so a sync regression (like Spec 1's grant-subscription bug) can't ship.

**Architecture:** One `targets.ts` registry drives both a Playwright `project` and a `webServer` per app. Apps are built by turbo (`e2e` task `dependsOn: ["^build"]`); Playwright only `start`/`preview`s the production build on a fixed port. Each project runs only its capability's spec via `testMatch`. Tests open the route **directly** (SSR) in two browser contexts and assert the live badge / toggle / comment propagation — plus that a `subscribe` WS frame was actually sent on load (the regression fingerprint).

**Tech Stack:** `@playwright/test` (Chromium only), Turborepo, pnpm workspaces. No app code changes.

**Design source:** `docs/superpowers/specs/2026-07-14-examples-sync-e2e-design.md` (read it first — especially "Load routes DIRECTLY", the ports table, and "Known noise").

**Conventions:**

- Prettier: 120 print width, double quotes, semicolons, trailing commas. All files must be prettier-clean (`pnpm format:check` covers them).
- Commit messages: no `Co-Authored-By` trailer.
- The e2e package defines **only** an `e2e` script (+ `test:ui`, `install-browsers`) — no `build`/`lint`/`test`/`check-types` scripts, so `turbo run lint|test|check-types` skip it automatically.

**Key facts (verified in Spec 1's survey):**

- pnpm workspace already globs `examples/*`, so `examples/e2e` is auto-included.
- Package names for `--filter`: `rxfy-example-next-blog`, `rxfy-example-rr7-blog`, `vite-blog-framework`, `rxfy-example-waku-blog`, `rxfy-template-vite`, `rxfy-template-next`.
- Ports 4301–4306; every app reads `PORT` from env **except waku**, which needs `waku start --port 4304` (WS pinned to 8090). All apps need `RXFY_SECRET` set.
- Confirmed selectors — blog: `a[href^="/posts/"]`, `input[placeholder="Your name"]`, `textarea[placeholder="Your comment…"]`, `button:has-text("Post comment")`, badge `button` matching `/new comment/i`. Todos: `input[placeholder="What needs doing?"]`, `button:has-text("Add")`, `button.updates-badge`, `li input[type=checkbox]`.
- `vite-blog-framework` and `templates/vite` emit a React #418 hydration warning — do NOT assert on page/console errors.

**Verification commands (from repo root):**

- Build all apps: `pnpm turbo build` (or `--filter` the six apps).
- Install browsers once: `pnpm --filter rxfy-e2e exec playwright install chromium`.
- Run one project: `pnpm --filter rxfy-e2e exec playwright test --project=template-vite`.
- Run all: `pnpm test:e2e`.

```ts
export type Capability = "sync-blog" | "sync-todos";

export type Target = {
  /** Playwright project name + baseURL host label. */
  name: string;
  /** pnpm --filter package name. */
  filter: string;
  capability: Capability;
  /** Fixed, unique HTTP port. */
  port: number;
  /** webServer command (reuses the app's own prod script). */
  command: string;
  /** Extra env for the server process. */
  env: Record<string, string>;
};

// Any non-empty secret works; it only has to match between the HTTP side and the WS server, which
// both read RXFY_SECRET from the same process env.
const RXFY_SECRET = process.env.RXFY_SECRET ?? "e2e-secret";

export const targets: Target[] = [
  {
    name: "next-blog",
    filter: "rxfy-example-next-blog",
    capability: "sync-blog",
    port: 4301,
    command: "pnpm --filter rxfy-example-next-blog start",

import { defineConfig, devices } from "@playwright/test";
import { targets } from "./targets";

const isCI = !!process.env.CI;

export default defineConfig({
  testDir: "./tests",
  // Tests within a project share one server; keep them serial to avoid racing on shared DB state.
  fullyParallel: false,
  workers: 1,
  forbidOnly: isCI,
  retries: isCI ? 2 : 0,
  reporter: isCI ? [["github"], ["list"]] : "list",
  timeout: 60_000,
  expect: { timeout: 10_000 },
  use: { trace: "on-first-retry" },

  // One project per app; each runs only its capability's spec.
  projects: targets.map((t) => ({
    name: t.name,
    testMatch: `**/${t.capability}.spec.ts`,
    use: { ...devices["Desktop Chrome"], baseURL: `http://localhost:${t.port}` },
    metadata: { capability: t.capability },
  })),

  // One production server per app, started once and reused across that project's tests.
  webServer: targets.map((t) => ({
    command: t.command,
    url: `http://localhost:${t.port}/`,
    env: t.env,
    reuseExistingServer: !isCI,
    timeout: 120_000,
    stdout: "pipe",
    stderr: "pipe",
  })),
});
```

```ts
import type { Browser, BrowserContext, Page } from "@playwright/test";

export type Tab = {
  context: BrowserContext;
  page: Page;
  /** Count of sync `subscribe` frames this tab sent over its /live WebSocket. */
  subscribeFrames: () => number;
};

/**
 * Open `url` in a fresh browser context (a "tab"), counting the sync `subscribe` frames sent on its
 * WebSocket. A direct SSR load that sends zero subscribe frames is the fingerprint of the
 * grant-subscription regression this suite guards against.
 */
export async function openTab(browser: Browser, url: string): Promise<Tab> {
  const context = await browser.newContext();
  const page = await context.newPage();
  let subscribeFrames = 0;
  page.on("websocket", (ws) => {
    ws.on("framesent", (frame) => {
      const payload = typeof frame.payload === "string" ? frame.payload : "";
      if (payload.includes('"subscribe"')) subscribeFrames += 1;
    });
  });
  await page.goto(url, { waitUntil: "networkidle" });
  return { context, page, subscribeFrames: () => subscribeFrames };
}

/**
 * Discover the first post URL from a blog app's home page so the test can load the post detail
 * DIRECTLY (SSR) — never by clicking through, which would mask the regression.
 */
export async function scoutFirstPostUrl(browser: Browser, baseURL: string): Promise<string> {
  const context = await browser.newContext();
  try {
    const page = await context.newPage();
    await page.goto(baseURL, { waitUntil: "networkidle" });
    const href = await page.locator('a[href^="/posts/"]').first().getAttribute("href");
    if (!href) throw new Error(`no post link found on ${baseURL}`);
    return new URL(href, baseURL).toString();
  } finally {
    await context.close();
  }
}
```

- [ ] **Step 1: Create `examples/e2e/tests/sync-todos.spec.ts`**

```ts
import { expect, test } from "@playwright/test";
import { todos } from "../fixtures/selectors";
import { openTab } from "../fixtures/two-tab";

test.describe("sync-todos", () => {
  test("create in one tab shows a live badge in another, then applies", async ({ browser, baseURL }) => {
    const a = await openTab(browser, baseURL!);
    const b = await openTab(browser, baseURL!);
    try {
      // Both SSR-loaded tabs must have subscribed on load; zero is the regression fingerprint.
      expect(a.subscribeFrames(), "tab A subscribed on direct load").toBeGreaterThan(0);
      expect(b.subscribeFrames(), "tab B subscribed on direct load").toBeGreaterThan(0);

      const title = `e2e-todo-${Date.now()}`;
      await b.page.fill(todos.addInput, title);
      await b.page.click(todos.addButton);

      const badge = a.page.locator(todos.badge);
      await expect(badge).toBeVisible();
      await badge.click();
      await expect(a.page.getByText(title)).toBeVisible();
    } finally {
      await a.context.close();
      await b.context.close();
    }
  });

  test("toggle propagates live to BOTH tabs, including the one that clicked", async ({ browser, baseURL }) => {
    const a = await openTab(browser, baseURL!);
    const b = await openTab(browser, baseURL!);
    try {
      const boxA = a.page.locator(todos.checkbox).first();
      const boxB = b.page.locator(todos.checkbox).first();
      const before = await boxB.isChecked();

      // Toggle in B. There is no optimistic update — B relies on the echo patch, same as A.
      await boxB.click();

      // The clicking tab (B) AND the other tab (A) must both reflect the new state.
      await expect(boxB, "clicking tab reflects its own toggle").toBeChecked({ checked: !before });
      await expect(boxA, "other tab reflects the toggle").toBeChecked({ checked: !before });
    } finally {
      await a.context.close();
      await b.context.close();
    }
  });
});
```

## Notes / risks

- **Six servers at once.** Playwright boots all six `webServer`s before running. If CI wall-time is a
  concern, split into two CI jobs by capability (`--project=` filters) — the registry makes this a
  one-line change. `log()` nothing is silently dropped; all six run.
- **Waku 8090 singleton.** Only the one waku project may bind 8090; ensure no stray process holds it.
- **Shared DB state.** `fullyParallel: false` + `workers: 1` keeps a project's tests serial so they
  don't race on the app's PGlite. Todos tests use unique titles and relative toggle assertions, so
  they're order-independent.
- **#418 hydration warning** in vite-blog-framework / templates/vite is expected noise — the specs
  never assert on page/console errors, only on sync behavior.
- **Depends on Spec 1** (PR #26). This branch is stacked on it so `templates/next` is already the
  sync-todos app; rebase onto `develop` once #26 merges (same `--onto` move used for Spec 1).
