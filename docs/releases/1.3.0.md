# Release 1.3.0

## Packages

| Package | Version |
| ------- | ------- |
| `rxfy` | `1.3.0` |
| `rxfy-react` | `1.3.0` |

## Highlights

`defineState` now supports plain (non-normalized) value fields alongside entity fields, so a state can mix normalized entities with primitives and plain objects that pass through `data$` unchanged. The `data$` / `set` / `setRaw` types are now derived directly from the field map, preserving each plain field's value type. `createModel` adopts a single config-object signature for consistency with the rest of the API.

## Changes

### New Features

- **Plain value fields in `defineState`** — a field entry may now be a bare zod schema (instead of `array()` / `single()`), declaring a plain value (boolean, primitive, or object). Plain fields live in the query state and pass through `data$` unchanged, distinct from entity fields that normalize into model stores. They are validated against their schema in development and passed through in production ([state.ts](../../packages/rxfy/src/state/state.ts), [normalize.ts](../../packages/rxfy/src/state/normalize.ts)).
- **Field-driven query/writable shapes** — `StateDescriptor`, `useStateData`, and `StateHandle` now carry `TQuery` / `TWritable` type parameters derived from the field map (`QueryShapeFromFields` / `WritableQueryShapeFromFields`), so `data$`, `set`, and `setRaw` keep the precise type of each plain field rather than collapsing to the entity-only shape ([state.ts](../../packages/rxfy/src/state/state.ts), [useStateData.ts](../../packages/rxfy-react/src/useStateData.ts)).
- **`isFieldDescriptor` guard** — a new exported runtime guard that distinguishes an entity descriptor (`array` / `single`) from a bare zod schema, used to discriminate field entries during normalization ([model.ts](../../packages/rxfy/src/model/model.ts)).

### Breaking Changes

- **`createModel` takes a single config object** — the schema has been merged into the options object to match the config-object shape used elsewhere (e.g. `useStateData`). Update call sites:

  ```ts
  // Before
  createModel(schema, { getKey, name });

  // After
  createModel({ schema, getKey, name });
  ```

  The exported `CreateModelConfig` type describes the new shape ([model.ts](../../packages/rxfy/src/model/model.ts)).
