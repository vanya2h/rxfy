## Highlights

rxfy 3.1 adds **model relations with per-state joins**: declare `ref()`/`refArray()` fields on a model and join them per fetch with `single(Model).with({ rel: true })` (Prisma-`include` style, recursive), so list and detail payloads share one normalized store. Joined reads are **view-typed** â€” a `StoreKey` minted by a joined query yields an entity whose joined relations are required (no `!`) â€” and the joins extend end to end into the **real-time sync** layer via a type-safe `fk` map. A batch of `z.infer`-style state inference helpers (`ParamsOf`, `ShapeOf`, `NormalizedOf`, â€¦) lets app code name a state's shapes without touching phantom carriers.

This release is additive. The one caveat is a **type-only** tightening of `ModelStore.get`, migrated with a single new `asKey` helper â€” see Breaking Changes.

## Changes

### New Features

- **Model relations with per-state joins** â€” declare relation fields in a model schema with `ref()`/`refArray()` and join them per fetch with `single(Model).with({ rel: true })`. A nested map (`{ rel: { sub: true } }`) joins recursively, so list and detail payloads normalize into one shared store. Relations infer as `StoreKey<R> | undefined` (`ref`) / `StoreKey<R>[] | undefined` (`refArray`) on the base entity, and become required on a joined view. New exports in `rxfy`: `ref`, `refArray`, `asKey`, plus the `StoreKey`, `EntityView`, `IncludeMap`, and `IncludeFor` types ([packages/rxfy/src/model/model.ts](packages/rxfy/src/model/model.ts), [packages/rxfy/src/model/model-store.ts](packages/rxfy/src/model/model-store.ts), [packages/rxfy/src/state/state.ts](packages/rxfy/src/state/state.ts), [packages/rxfy/src/state/normalize.ts](packages/rxfy/src/state/normalize.ts)).

- **View-typed joined reads** â€” `ModelStore.get` preserves the key's brand: a `StoreKey` minted by a joined query shape (`single(Post).with({ author: true, comments: { author: true } })`) yields an entity whose joined relations are **required** and read without a `!`. Thread the branded keys down to child components (instead of raw ids) and each `get` returns the matching view; a base `StoreKey<T>` still yields the base entity. Enabled by making the `StoreKey` phantom brand covariant, so a joined-view key is assignable wherever the base key is expected.

- **`useModelStoreValue(model, id)` (`rxfy-react`)** â€” a non-throwing reactive read for components that may render whether or not a relation was joined. Complements `useModelStore` / `store.get`, which throw on an unloaded key ([packages/rxfy-react/src/useModelStoreValue.ts](packages/rxfy-react/src/useModelStoreValue.ts)).

- **Real-time sync for model relations (`rxfy-server` / `rxfy-client`)** â€” `createModel` accepts a type-safe `fk` map (`{ category: "categoryId" }`, both sides inferred from the schema) linking each relation to its foreign-key column. `sync.serve` now recurses joined relations, so the signed grant enumerates nested entity topics (client-fetch and SSR) and the served payload keeps nested entities for the client to normalize. The client patch handler mirrors a relation's id from its `fk` column, so a flat patch keeps the relation resolvable. Additive â€” store-only and non-joined apps are unaffected ([packages/rxfy-server/src/sync.relations.test.ts](packages/rxfy-server/src/sync.relations.test.ts), [packages/rxfy-client/src/sync-client.ts](packages/rxfy-client/src/sync-client.ts)).

- **`registry.descriptor(name)` accessor** â€” resolve a registered model's full descriptor by name from the model registry.

- **State inference helper types (`rxfy`)** â€” name a state's shapes without reaching into the phantom `_shape`/`_query`/`_shapeInput` carriers. From a `defineState` value: `ParamsOf<S>`, `ShapeOf<S>` (denormalized output), `NormalizedOf<S>` (the normalized id shape `data$` emits), `WritableOf<S>`, and `InputOf<S>` (the denormalized input a fetch/serve payload has before parsing). Plus `ViewOf<Key>` to deref a `StoreKey` to the entity view it was minted for. Mirrors `z.infer` for schemas â€” e.g. `NormalizedOf<typeof myState>["post"]` instead of `NonNullable<(typeof myState)["_query"]>["post"]`.

### Breaking Changes

- **Type-only: `ModelStore.get` now requires a `StoreKey<T>`, not a bare string.** Query shapes mint `StoreKey`s automatically, so ids read from a state's `data$` keep working unchanged. A **raw string passed directly to `get()` no longer type-checks** â€” migrate it with the new `asKey` helper. This is a compile-time change only; runtime behavior is identical, which is why it ships as a minor (on-pattern code sourcing ids from query shapes is unaffected).

  ```ts
  // before
  store.get(id);
  // after â€” brand a genuinely-raw id (e.g. a URL param) into the keyspace
  store.get(asKey(Model, id));
  ```

### Dependencies

- No production dependency changes. `rxfy-react` gains `useModelStoreValue`; the sync-relations feature spans `rxfy`, `rxfy-server`, and `rxfy-client` (already linked in the fixed version group).
