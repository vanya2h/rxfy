# Release 1.2.0

## Packages

| Package | Version |
| ------- | ------- |
| `rxfy` | `1.2.0` |
| `rxfy-react` | `1.2.0` |

## Highlights

This release makes paginated and infinite-scroll lists first-class in `rxfy-react`: a new `useStatePagedData` hook keeps page 0 SSR-friendly while `loadMore()` fetches and O(page-size) appends later pages. `useStateData`'s `setRaw` now accepts denormalized entity objects and normalizes them on write, retiring the "entity not loaded" footgun, and `reload()` now refetches in place so every subscriber to a keyed state revalidates together on a stable `data$`. `useStateData` also adopts a single config-object signature — the one breaking change in this release.

## Changes

### New Features

- **`useStatePagedData` — a focused hook for paginated / infinite-scroll lists of a single entity type.** You give it a `model` (the list is always `array(model)`) and a `key`; `data$` emits a flat `string[]` of ids. Page 0 is SSR'd and hydrated through `useStateData`, while `loadMore()` fetches and appends later pages via a pluggable `getCursor` and `select`, with built-in `isLoading` and `hasMore`. Appending is O(page size) — only the new page's entities are written, never the whole list. Exports `PagedListHandle` and `UseStatePagedDataConfig`. ([`packages/rxfy-react/src/useStatePagedData.ts`](../../packages/rxfy-react/src/useStatePagedData.ts))

- **`setRaw` now accepts denormalized entities.** `setRaw` accepts denormalized entity objects (or a mix of ids and entities) in model-field slots and normalizes them on write — appending a page no longer needs a manual `normalizeResult` call. Object elements are written to their model stores (schema-validated in development); string ids pass through unchanged, so existing id-only `setRaw` calls are unaffected. The updater form still receives `prev` as ids, keeping appends O(page size). ([`packages/rxfy-react/src/useStateData.ts`](../../packages/rxfy-react/src/useStateData.ts))

- **`normalizeWritable` helper and `WritableQueryShapeOf` type added to `rxfy`.** Supporting primitives for writing a mixed id/entity shape into the normalized stores, exported from the package barrel. ([`packages/rxfy/src/state/normalize.ts`](../../packages/rxfy/src/state/normalize.ts))

- **More stable `data$` in `useStateData`.** `reload()` now refetches in place — it flips the shared query atom to PENDING and refetches into it instead of deleting the cache entry and rebuilding the handle, so every component subscribed to the same keyed state sees the refreshed result (previously only the caller did) and `data$` keeps a stable identity across a reload. `data$` identity is now stable across re-renders, a changing `defaultData`, and an identity-unstable-but-value-equal `params` (the query is keyed by the params _value_). `set` / `setRaw` abort any in-flight fetch before committing FULFILLED, so an explicit write can't be clobbered by a late-arriving fetch result. `useStatePagedData.reload()` resets its own pagination state to match. ([`packages/rxfy-react/src/useStateData.ts`](../../packages/rxfy-react/src/useStateData.ts))

### Breaking Changes

- **`useStateData` now takes a single config object instead of positional arguments.** Replace `useStateData(state, fetchFn, params, { defaultData })` with `useStateData({ state, fetchFn, params, defaultData })`. This matches the shape of `useStatePagedData` and makes the optional `defaultData` a flat field rather than a separate options argument. Also exports the `UseStateDataConfig` and `Updater<T>` types — `Updater<T>` (`T | ((prev: T) => T)`) is the `useState`-style setter union used by `set` and `setRaw`. ([`packages/rxfy-react/src/useStateData.ts`](../../packages/rxfy-react/src/useStateData.ts))
