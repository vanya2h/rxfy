# Release 1.1.0

## Packages

| Package      | Version |
| ------------ | ------- |
| `rxfy`       | `1.1.0` |
| `rxfy-react` | `1.1.0` |

## Highlights

This release upgrades the `zod` peer dependency to v4, which is required for all consumers. Alongside the upgrade, unused production dependencies have been removed from `rxfy` and internal type signatures have been updated to match zod v4's revised `ZodType` generics.

## Changes

### Bug Fixes

- **ZodType generics updated for zod v4 compatibility** — `ModelDescriptor`, `createModel`, `StateDescriptor`, and `defineState` now use the two-argument `z.ZodType<TOutput, TInput>` form instead of the three-argument form with `z.ZodTypeDef`. This aligns with zod v4's changed generic signature and preserves correct inference for schemas with distinct input and output types (e.g. branded ids). ([`packages/rxfy/src/model/model.ts`](../../packages/rxfy/src/model/model.ts), [`packages/rxfy/src/state/state.ts`](../../packages/rxfy/src/state/state.ts))

### Dependencies

- **`zod` peer dependency upgraded to `^4.0.0`** in both `rxfy` and `rxfy-react`. Consumers must upgrade zod to v4.
- **Removed unused production dependencies** `p-queue` and `object-hash` from `rxfy`. These were not used in any published code path.
- **Removed `rxfy` from `rxfy-react` peer dependencies** to prevent spurious major version bumps in RC release cycles.
