# Release 1.0.5

## Packages

| Package      | Version |
| ------------ | ------- |
| `rxfy`       | `1.0.5` |
| `rxfy-react` | `1.0.5` |

## Highlights

`useStateData` now accepts a `defaultData` option that seeds state from pre-fetched data (e.g. a React Router loader) without triggering a fetch on first load. This makes it straightforward to pass server-fetched data directly into the reactive layer while keeping subsequent navigations fully reactive.

## Changes

### New Features

- **`defaultData` option for `useStateData`** (`packages/rxfy-react/src/useStateData.ts`) — pass `{ defaultData: TShape }` as the fourth argument to pre-populate state from an external source (loader, SSR prop, cache). The atom is seeded with a `FULFILLED` value only when its current status is `IDLE`, so it does not override in-flight or already-resolved state. Entities in `defaultData` are normalized into the model stores exactly as they would be after a live fetch. A new exported type `UseStateDataOptions<TShape>` accompanies this option.
