## Highlights

rxfy 3.0 reworks the real-time stack end to end. The "live" surface is renamed **sync** across every package; the server is decoupled from Drizzle behind a `SyncStorage` port (with new `rxfy-server-drizzle` and `rxfy-server-memory` adapters); and real-time authorization moves to **signed channel + entity grants** so subscriptions are automatic and entity ids no longer need to be unguessable. Two new packages ship — `rxfy-client`, the framework-agnostic browser half of the sync stack, and `create-rxfy-app`, a scaffolder for standalone rxfy apps — alongside a batch of core tightening (`createModel` name and `defineState` key now required, `ModelStore.get` returns a writable atom, a typed model registry).

This is a major, breaking release. Migration notes are inline with each breaking change below.

## Changes

### New Features

- **`create-rxfy-app` (new package)** — scaffold a standalone rxfy app from an official template via `pnpm create rxfy-app`. Ships three templates: `vite-spa` (client-only Vite + React SPA, the simplest setup), `vite` (fully SSR'd live todos: Vite + React Router + Hono + Drizzle/PGlite + sync over WebSocket), and `next` (Next.js App Router seeded from React Server Components — RSC prefetch + hydrate, isomorphic fetch, server actions). Built on incur: interactive clack prompts in a terminal, plus a structured JSON/TOON envelope with stable error codes (`DIR_NOT_EMPTY`, `UNKNOWN_TEMPLATE`, …) when driven by agents or pipes. Includes built-in `skills add`, `--llms`, and `--mcp` agent integrations. Requires Node 22+.

- **`rxfy-client` (new package)** — the framework-agnostic browser half of the sync stack. `createSyncClient({ registry, transport, renewUrl? })` takes custody of the signed channel grants the data delivered (each `$grant` lifted by `useStateData`, plus the SSR `grants` via `readSsrGrants()`), subscribes over the WebSocket transport, renews grants ahead of expiry through an app-mounted `renewUrl` endpoint, and replays its whole grant set on every reconnect. Sync updates no longer require React; `rxfy-react` re-exports `createSyncClient` and `readSsrGrants`.

- **`rxfy-server-drizzle` / `rxfy-server-memory` (new adapter packages)** — storage adapters for the now storage-agnostic sync server. `rxfy-server-drizzle` exports `defineResource`, `drizzleStorage`, and `DrizzleBinding`; `rxfy-server-memory` exports `defineCollection`, `memoryStorage`, and `MemoryBinding`. Apps that only need stale-notification plumbing (e.g. an in-memory store publishing `stale` on writes) can now wire sync without pulling in the Drizzle peers.

- **Signed channel + entity grants** — automatic real-time subscriptions with no declared-grant flow. `sync.serve(state, params, data)` signs a per-state JWT grant (channel + expiry + the exact entity `name:id` topics in the payload) and attaches it as `$grant`; `useStateData` lifts it automatically and subscribes. Because the grant names a fixed, signed set of entity topics, entity ids no longer need to be unguessable. New exports: `collectShapeTopics` and `collectEntityTopics` in `rxfy`, `grantsHydration` in `rxfy-server` for one-call SSR grant signing.

- **`rxfy-server/hub` — Drizzle-free subpath** — the socket-keyed in-memory hub, subscription-id helpers (`entitySubscription`/`channelSubscription`/`entityTopicSubscription`), channel derivation (`touch`/`invalidationChannel`/`StateChannelDescriptor`), grant primitives (`signGrant`/`verifyGrant`), and the `grantsHydration(registry, { secret })` SSR helper — usable without the Drizzle peer dependency. (In this release the `hub` and `browser` subpaths collapse into the single storage-agnostic entry; see Breaking Changes.)

- **Typed model registry** — `IModelRegistry` is now generic over its registered models: `createModelRegistry(postModel).add(commentModel)` types the registry as `IModelRegistry<{ post; comment }>`. `registry.store("post")` is a typed lookup returning `ModelStore<Post>`; `registry.model(descriptor)` only accepts registered descriptors at compile time; `stashHydration` and `namedStores` are name-checked. The closed set is a compile-time guard only — runtime is unchanged, and bare `IModelRegistry` / no-arg `createModelRegistry()` remain the open registry. New exports `AnyModelDescriptor` and `ModelsShape`; `createModel` captures its `name` as a literal type.

- **`sync.serve` parses the input shape** — `serve(state, params, data)` now accepts the state's _input_ shape and parses it through the field schemas. Raw DB rows (unbranded ids, extra columns like `createdAt`) go in with no casts; the returned payload has ids branded and unknown keys stripped. rxfy threads the zod Input type through `ModelDescriptor`, `FieldDescriptor`, and `StateDescriptor` (defaulted, non-breaking), with new `InputShapeFromFields` and `parseShape(fields, input)` helpers.

- **Typed `window` param keys** — `defineState`'s `window` option is now typed as `readonly (keyof TParams & string)[]`, so a typo'd window entry is a compile error instead of a silently ignored dimension.

### Breaking Changes

- **"live" is renamed "sync" across the stack.** Client: `createLiveClient` → `createSyncClient`, `useLiveClient` → `useSyncClient`, `StoreProvider`'s `liveClient` prop → `syncClient`, and types `LiveClient` / `LiveClientConfig` / `LiveTransport` → `SyncClient` / `SyncClientConfig` / `SyncTransport`. Server: `createLive` → `createSync`, and types `Live` → `Sync`, `LiveConfig` → `SyncConfig`, `LiveStorage` → `SyncStorage`. Migration: rename the imports/props above; React apps continue importing `createSyncClient`/`readSsrGrants` from `rxfy-react` (re-exported from `rxfy-client`).

- **Server decoupled from Drizzle behind a storage port.** `createServer({ db, resources, hub, secret })` → `createSync({ storage: drizzleStorage(db), hub, secret })`. `Resource` is now neutral (carries an opaque adapter binding) and `Sync` is generic over the binding. Import `defineResource` from `rxfy-server-drizzle` (was `rxfy-server/browser`); `createResourceRegistry` stays in `rxfy-server`. The `rxfy-server/hub` and `rxfy-server/browser` subpaths collapse into the single Drizzle-free `rxfy-server` entry.

- **Declared grants removed; grants are signed and automatic.** `createServer` (now `createSync`) requires a `secret`; `serve` returns the parsed shape plus `$grant`; new `renew`. The removed surface: `createTopicKeyer`, `grant`, `GrantSpec`, `Grants` (rxfy-server); `addGrants` and grant props (rxfy-react); `grantsHydration`'s `secret`/`ttlMs` options (SSR now reuses the served grant verbatim). The wire protocol is v2: `subscribe { grant, entities }` is the only client frame — hashed-token subscribe/unsubscribe frames are gone; `rxfy-ws`'s client transport is now `send`/`onOpen`. **Security:** keep `Cache-Control: private, no-store` on state endpoints as response hygiene (the payload carries a bearer grant).

- **`createModel` now requires `name`** (and `ModelDescriptor.name` is non-optional). The name is the model's stable identity for SSR dehydration and sync topics; making it mandatory removes every unnamed-model fallback (`dehydrate` no longer skips stores, `modelTopic` no longer throws, `added$` covers every store). Migration: add `name: "..."` to any `createModel` call that omitted it — names must be unique per app.

- **`defineState` now requires `key`** (and `StateDescriptor.key` is a required `string`). Every state participates in the SSR query cache and derives a sync invalidation channel; the keyless opt-out is gone. Keyed descriptors are now directly assignable to `StateChannelDescriptor`, so `touch(postsState, params)` works without a cast. Migration: add a unique `key` to any `defineState` call that omitted one.

- **`ModelStore.get(key)` now returns a writable `IAtom<T>`** (the former `entity(key)` handle) instead of a filtering `Observable<T>`, and `ModelStore.entity` is removed. Accessing a key that has never been `set` **throws** instead of waiting silently — ids are expected to come from fulfilled states, so an unloaded access is a programming error surfaced early. `get(id)` reads synchronously (`.get()`), subscribes reactively, and writes back (`.set()`/`.modify()`); results are no longer sync-marked (no `usePending`/`<Pending>` probe). Migration:
  - `store.entity(id)` → `store.get(id)` (same semantics).
  - `<Pending value$={store.get(id)}>{(x) => …}</Pending>` → `const [x] = useAtom(useMemo(() => store.get(id), [store, id]))` and render directly.
  - To probe for presence without throwing, use `store.getValue(id)`.

- **`sync.serve` changes behavior** — because it now parses the input shape, the result is a new parsed object (not the same reference), and invalid data throws.

- **Writer signatures tightened (`rxfy-server`).** `sync.update`/`sync.create` infer the resource's `TRow` instead of erasing it, so resources carrying an injected (branded/narrower) model fit without an `as unknown as Resource<typeof table>` cast. `sync.create` no longer types `undefined` in its result (a zero-row `.returning()` now throws); `sync.update` resolving `undefined` is the documented not-found contract, and a not-found update no longer publishes its `touch` targets.

### Dependencies

- New peer/adapter split: Drizzle now lives in **rxfy-server-drizzle** (peers on `drizzle-orm` / `drizzle-zod`); `rxfy-server` core no longer requires the Drizzle peers.
- `rxfy-client` is a new dependency of the sync path; `rxfy-react` depends on it and re-exports its client API.
- `create-rxfy-app` requires **Node 22+**.
